# 중고거래 플랫폼 개발 체크리스트 현황

기준 일자: 2026-07-23

## 분류 기준

- 완료: 명세 요구사항이 코드에 반영되어 있고 기본 동작이 확인 가능한 상태
- 부분완료: 핵심 흐름은 있으나 보안 조건, 예외 처리, 데이터 모델, 운영 요건이 부족한 상태
- 미구현: 명세상 요구가 있으나 현재 코드에 구현이 없거나 골격만 있는 상태

## 현재 구현 분류표

| 영역 | 상태 | 근거 | 다음 액션 |
| --- | --- | --- | --- |
| 회원가입 | 완료 | 이메일/아이디/비밀번호 검증과 중복 체크 반영 | 테스트 유지보수 |
| 로그인/로그아웃 | 완료 | 실패 누적, 잠금, 비활성 계정 차단, 공통 오류 응답 반영 | 경고 정리 |
| CSRF | 부분완료 | `Flask-WTF`, 토큰 발급, 주요 POST/PATCH/DELETE 보호 반영 | 프론트 전체 토큰 사용 일관화 |
| 프로필/마이페이지 | 부분완료 | 내 정보 조회, 프로필 수정, 비밀번호 변경, 잔액 조회 반영 | 화면/UX 정리 |
| 상품 등록/조회/수정/삭제 | 부분완료 | CRUD, 소프트 삭제, 가격/길이 검증, 업로드 보안 강화 반영 | 상태 전이 규칙과 관리자 제어 고도화 |
| 상품 검색/페이징 | 완료 | 목록 조회, 검색어, 페이지네이션 구현 | 응답 스키마 정리 |
| 전체 채팅 | 부분완료 | 전체 채팅 조회/등록, 길이 제한, rate limit 반영 | 운영 로그와 추가 정책 보강 |
| 1:1 채팅 | 부분완료 | 채팅방 생성/조회, 메시지 조회/등록, 길이 제한, rate limit 반영 | 거래 상태 연동 고도화 |
| 신고 | 부분완료 | 사용자/상품 신고, 중복 신고 방지, 자동 차단, 사유 검증, 상태 모델 반영 | 운영 정책 세분화 |
| 자동 차단/휴면 처리 | 부분완료 | 기준치 설정, 사용자 비활성화, 재로그인 차단 반영 | 기존 세션 즉시 종료 보강 |
| 관리자 기능 | 부분완료 | 신고 목록, 사용자/상품 상태 변경, 신고 검토 상태, 감사 로그 조회, 기준 설정 반영 | 운영 대시보드 보강 |
| 결제/송금 | 부분완료 | 잔액 기반 송금, 검증, 감사 로그 반영 | 거래 플로우와 정산 상태 연동 |
| 오류 처리 | 완료 | 공통 401/403/404/500 응답 규격 반영 | 세부 메시지 다듬기 |
| 로그/감사 | 부분완료 | 관리자 조치, 삭제, 송금, 기준 변경 로그 반영 | 로그인 성공/실패, 채팅/신고 운영 로그 확대 |
| 서버/배포 보안 | 부분완료 | 쿠키 속성, 세션 만료, 세션 버전 반영 | 운영 환경 분리와 배포 설정 보강 |
| 테스트 | 부분완료 | `pytest` 실행 환경 정리, 현재 기준 테스트 통과 | 레거시 인코딩 테스트 제거 및 경고 정리 |

## 보안 우선순위

### P0

| 항목 | 이유 |
| --- | --- |
| 회원가입 입력 검증 강화 | 취약한 입력 검증은 계정 데이터 오염과 후속 취약점의 시작점 |
| 로그인 실패 제한 및 계정 잠금 | 무차별 대입 공격 방어에 직접 연결 |
| 비활성 계정 로그인 차단 | 신고 기반 제재가 실제로 효력을 가지려면 필수 |
| 세션/쿠키 기본 보안 설정 | 인증 전반에 공통으로 영향을 줌 |
| 공통 인증/인가 오류 응답 정리 | 이후 기능 개발과 보안 테스트의 기준선 역할 |

### P1

| 항목 | 이유 |
| --- | --- |
| 상품 입력 검증과 파일 업로드 보안 | 파일 업로드와 상품 등록은 공격면이 넓음 |
| 채팅 길이 제한 및 속도 제한 | 저장형 XSS, 도배, 자원 남용 방지 |
| 신고 사유 검증 및 상태 모델 | 운영 정책과 관리자 기능의 핵심 |

### P2

| 항목 | 이유 |
| --- | --- |
| 관리자 검토 흐름 고도화 | 운영 완성도 향상 |
| 결제/송금 실구현 | 범위가 크고 선행 보안 정비 후 진행하는 편이 안전 |
| 테스트 체계 정비 | 기능 안정화 이후 회귀 방지 |

## 현재 남은 핵심 항목

1. 상품/거래 상태 전이 규칙을 더 촘촘하게 정리
2. 기존 세션 즉시 종료를 더 강하게 보장하는 구조 보강
3. 로그인 성공/실패와 운영 이벤트에 대한 감사 로그 확대
4. 결제/송금을 거래 완료 흐름과 연결
5. 레거시 인코딩 테스트 제거와 SQLAlchemy/Flask 경고 정리
