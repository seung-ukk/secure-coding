1. 프로젝트 개요
1.1 프로젝트명
중고거래 및 사용자 소통 플랫폼
1.2 프로젝트 목적
사용자가 회원가입과 로그인을 통해 플랫폼을 이용하고, 중고 상품을 등록·조회·관리할 수 있는 웹 서비스를 개발한다.
사용자는 전체 채팅과 1대1 채팅을 통해 다른 사용자와 소통할 수 있으며, 불량 사용자 또는 부적절한 상품을 신고할 수
있다.
신고가 일정 횟수 이상 누적된 상품은 차단 또는 삭제하고, 신고가 누적된 사용자는 휴면 계정으로 전환하여 안전한 플
랫폼 이용 환경을 제공한다.
1.3 주요 개발 목표
사용자 회원가입 및 로그인 기능 구현
사용자 프로필 및 마이페이지 기능 구현
상품 등록, 조회, 수정, 삭제 기능 구현
전체 채팅 및 1대1 채팅 기능 구현
사용자 및 상품 신고 기능 구현
신고 누적에 따른 상품 차단 및 사용자 휴면 처리
사용자 정보와 상품 정보를 데이터베이스로 관리
시큐어 코딩 원칙을 적용하여 주요 웹 취약점 예방

기능 설명
로그인한 모든 사용자가 참여할 수 있는 공용 채팅방을 제공한다.
표시 항목
작성자 계정명
메시지 내용
작성 시간
처리 조건
로그인한 사용자만 메시지를 작성할 수 있다.
메시지 최대 길이를 제한한다.
HTML 및 JavaScript 코드가 실행되지 않도록 처리한다.
일정 시간 동안 전송 가능한 메시지 수를 제한한다.
부적절한 메시지를 신고할 수 있도록 확장할 수 있다.
휴면 사용자는 채팅 기능을 사용할 수 없다.
5.2 1대1 채팅 기능
기능 설명
두 명의 사용자가 개별 채팅방을 통해 소통한다.
처리 조건
로그인한 사용자만 이용할 수 있다.
채팅 참여자만 해당 채팅방의 메시지를 조회할 수 있다.
URL의 채팅방 번호를 변경하여 다른 사용자의 채팅을 조회할 수 없어야 한다.
채팅방 생성 시 참여자 정보를 서버에서 검증한다.
자신과의 채팅방 생성을 제한할 수 있다.
차단 또는 휴면 상태인 사용자는 새 채팅을 생성하지 못하도록 한다.

6.2 불량 상품 차단 기능
기능 설명
상품 신고가 일정 횟수 이상 누적되면 해당 상품을 차단한다.
처리 조건
신고 횟수 기준은 관리자가 설정한다.
동일 사용자의 중복 신고는 신고 횟수에 한 번만 반영한다.
기준 횟수 이상 신고된 상품은 차단 상태로 변경한다.

관리자가 신고 내용을 검토한 후 삭제 여부를 결정할 수 있다.
자동 차단과 관리자 검토 상태를 구분할 수 있다.

9. 비기능 요구사항
9.1 보안성
모든 비밀번호는 안전한 알고리즘으로 해시하여 저장한다.
SQL Injection을 방지하기 위해 ORM 또는 Prepared Statement를 사용한다.
모든 사용자 입력값을 서버에서 검증한다.
출력값을 이스케이프하여 XSS를 방지한다.
상태 변경 요청에는 CSRF 방어를 적용한다.
인증이 필요한 기능에는 서버 측 인증 검사를 적용한다.
객체별 접근 권한을 서버에서 검사한다.
파일 업로드 기능에는 확장자, MIME 형식, 파일 크기 검사를 적용한다.
중요 작업과 관리자 작업을 로그로 남긴다.

15. 완료 기준
다음 조건을 모두 만족하면 프로젝트의 주요 기능이 완료된 것으로 판단한다.
회원가입 시 아이디 중복이 방지된다.
비밀번호가 안전하게 해시되어 저장된다.
로그인과 로그아웃이 정상적으로 동작한다.
사용자가 자신의 소개글과 비밀번호를 수정할 수 있다.
로그인한 사용자가 상품을 등록할 수 있다.
등록된 상품을 누구나 조회할 수 있다.
상품 등록자만 자신의 상품을 수정하거나 삭제할 수 있다.
상품 목록에서 상품을 선택하면 상세 페이지가 표시된다.
사용자가 전체 채팅과 1대1 채팅을 이용할 수 있다.
채팅방 참여자가 아닌 사용자는 대화 내용을 볼 수 없다.
상품과 사용자를 신고할 수 있다.
동일 사용자의 중복 신고가 방지된다.
신고 기준 이상인 상품은 차단된다.
신고 기준 이상인 사용자는 휴면 처리된다.
SQL Injection, XSS, CSRF 및 접근 권한 취약점에 대한 방어가 적용된다.
상품 이미지 업로드에 대한 보안 검사가 적용된다.
관리자와 주요 사용자 활동이 로그로 기록된다.
